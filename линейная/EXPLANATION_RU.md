## Подробное объяснение проекта ft_linear_regression (на русском)

В этом проекте мы учимся строить **простую линейную регрессию**: по пробегу машины (в километрах) предсказывать её цену.

### 1. Данные

Входной файл `data.csv` содержит реальные наблюдения:

```text
km,price
240000,3650
139800,3800
...
```

- **`km`** — пробег машины (фича, входной признак \(x\)).
- **`price`** — цена машины (целевая переменная \(y\)).

Каждая строка — это одна машина: сколько она прошла и за сколько продаётся.

### 2. Модель: что мы хотим найти

Мы предполагаем, что цена примерно линейно зависит от пробега:

\[
\text{estimatePrice}(mileage) = \theta_0 + \theta_1 \cdot mileage
\]

Где:

- \(\theta_0\) — цена машины с нулевым пробегом (пересечение прямой с осью \(y\)).
- \(\theta_1\) — насколько **меняется** цена, если пробег увеличится на 1 км (наклон прямой).

**Задача обучения**: подобрать такие \(\theta_0\) и \(\theta_1\), чтобы прямая как можно лучше приближала все точки из `data.csv`.

### 3. Ошибка (стоимость) и градиентный спуск

Для каждой машины с пробегом \(x_i\) и ценой \(y_i\) мы считаем предсказание:

\[
\hat{y}_i = \theta_0 + \theta_1 \cdot x_i
\]

Ошибка на этой точке:

\[
e_i = \hat{y}_i - y_i
\]

Чтобы оценить, насколько «плоха» вся модель, суммируем ошибки по всем \(m\) примерам.
В классическом варианте часто берут среднеквадратичную ошибку, но в задании даны формулы для **средней невзвешенной суммы ошибок** по каждой переменной:

\[
\text{tmp}\theta_0 = \text{learningRate} \cdot \frac{1}{m} \sum_{i=0}^{m-1} (\hat{y}_i - y_i)
\]
\[
\text{tmp}\theta_1 = \text{learningRate} \cdot \frac{1}{m} \sum_{i=0}^{m-1} (\hat{y}_i - y_i) \cdot x_i
\]

Затем делаем **шаг градиентного спуска**:

\[
\theta_0 \leftarrow \theta_0 - \text{tmp}\theta_0
\]
\[
\theta_1 \leftarrow \theta_1 - \text{tmp}\theta_1
\]

Важно: \(\theta_0\) и \(\theta_1\) нужно обновлять **одновременно**, используя временные значения.

### 4. Маленький числовой пример

Допустим, у нас всего две машины:

- Машина A: \(x_0 = 10\,000\) км, \(y_0 = 10\,000\) \$.
- Машина B: \(x_1 = 20\,000\) км, \(y_1 = 8\,000\) \$.

Пусть:

- начальные параметры: \(\theta_0 = 0\), \(\theta_1 = 0\),
- скорость обучения: \(\text{learningRate} = 0.000001\),
- количество примеров: \(m = 2\).

**Шаг 1. Считаем предсказания:**

\[
\hat{y}_0 = 0 + 0 \cdot 10\,000 = 0
\]
\[
\hat{y}_1 = 0 + 0 \cdot 20\,000 = 0
\]

**Ошибки:**

\[
e_0 = \hat{y}_0 - y_0 = 0 - 10\,000 = -10\,000
\]
\[
e_1 = \hat{y}_1 - y_1 = 0 - 8\,000 = -8\,000
\]

**Суммы:**

\[
\sum e_i = -10\,000 + (-8\,000) = -18\,000
\]
\[
\sum e_i \cdot x_i = (-10\,000)\cdot 10\,000 + (-8\,000)\cdot 20\,000
= -10^8 - 1.6\cdot 10^8 = -2.6\cdot 10^8
\]

**Временные значения:**

\[
\text{tmp}\theta_0 = 0.000001 \cdot \frac{1}{2} \cdot (-18\,000) = -0.009
\]
\[
\text{tmp}\theta_1 = 0.000001 \cdot \frac{1}{2} \cdot (-260\,000\,000) = -130
\]

**Обновление параметров:**

\[
\theta_0 \leftarrow 0 - (-0.009) = 0.009
\]
\[
\theta_1 \leftarrow 0 - (-130) = 130
\]

После одного шага модель стала:

\[
\hat{y} = 0.009 + 130 \cdot x
\]

Дальше повторяем шаги много раз, и \(\theta_0\), \(\theta_1\) постепенно сходятся к значениям, которые лучше всего описывают данные.

### 5. Нормализация признака (зачем она нужна)

В нашем коде пробег сначала **нормализуется** в диапазон \([0, 1]\). Это делается в функции `_normalize_feature`:

\[
x_{\text{norm}} = \frac{x - \min(x)}{\max(x) - \min(x)}
\]

Зачем это нужно:

- Пробеги имеют большой масштаб (десятки тысяч км).
- При больших числах шаги градиентного спуска могут быть слишком резкими или, наоборот, слишком маленькими — обучение будет нестабильным или медленным.

Поэтому:

1. Мы **обучаемся** на нормализованном \(x_{\text{norm}}\) и находим параметры \(a\) и \(b\) для модели
   \(\hat{y} = a + b \cdot x_{\text{norm}}\).
2. Потом **пересчитываем** их обратно в параметры \(\theta_0\) и \(\theta_1\) для исходного масштаба пробега:

   \[
   x_{\text{norm}} = \frac{x - \min}{\text{span}}, \quad \text{span} = \max - \min
   \]
   \[
   \hat{y} = a + b \cdot \frac{x - \min}{\text{span}}
   = \left(a - b \cdot \frac{\min}{\text{span}}\right) + \left(\frac{b}{\text{span}}\right) x
   \]

   Отсюда:

   \[
   \theta_1 = \frac{b}{\text{span}}, \quad
   \theta_0 = a - b \cdot \frac{\min}{\text{span}}
   \]

Таким образом, **пользователь и программа `predict.py` всегда работают с исходным пробегом в километрах**, а нормализация — это внутренний технический шаг для устойчивого обучения.

### 6. Структура кода в проекте

#### 6.1. `linear_regression.py`

Основные функции:

- **`load_data(path)`** — читает `data.csv`, достаёт столбцы `km` и `price`, приводит их к `float`.
- **`_normalize_feature(xs)`** — нормализует список значений в диапазон \([0, 1]\), возвращает нормализованные данные и значения \(\min\) и \(\max\).
- **`_gradient_descent(xs, ys, learning_rate, iterations)`** — реализует цикл градиентного спуска по формулам из задания, возвращает параметры \(a\) и \(b\) для нормализованного признака.
- **`train_from_csv(data_path, learning_rate, iterations)`**:
  - читает данные;
  - нормализует пробег;
  - запускает градиентный спуск;
  - пересчитывает \(a, b\) в \(\theta_0, \theta_1\) для исходного масштаба;
  - возвращает \(\theta_0, \theta_1\).
- **`estimate_price(mileage, theta0, theta1)`** — реализация формулы
  \(\text{estimatePrice}(mileage) = \theta_0 + \theta_1 \cdot mileage\).
- **`save_model(theta0, theta1, path="theta.csv")`** — сохраняет теты в CSV:

  ```text
  theta0,theta1
  8500.1234,-0.0289
  ```

- **`load_model(path="theta.csv")`** — читает \(\theta_0\), \(\theta_1\) из файла. Если файла нет или он некорректный, возвращает `0.0, 0.0` (как предписано в subject: до обучения теты = 0).

#### 6.2. `train.py`

Скрипт обучения:

```python
from linear_regression import train_from_csv, save_model


def main() -> None:
    theta0, theta1 = train_from_csv("data.csv", learning_rate=0.1, iterations=1000)
    save_model(theta0, theta1)
    print(f"Training complete.")
    print(f"theta0 = {theta0:.4f}")
    print(f"theta1 = {theta1:.8f}")


if __name__ == "__main__":
    main()
```

- Считает данные.
- Запускает градиентный спуск.
- Печатает найденные \(\theta_0\), \(\theta_1\).
- Сохраняет их в `theta.csv`.

#### 6.3. `predict.py`

Скрипт предсказания:

```python
from linear_regression import load_model, estimate_price


def main() -> None:
    theta0, theta1 = load_model()

    try:
        user_input = input("Enter car mileage (km): ").strip()
    except EOFError:
        print("No input provided.")
        return

    if not user_input:
        print("No mileage provided.")
        return

    try:
        mileage = float(user_input)
    except ValueError:
        print("Invalid mileage, please enter a number.")
        return

    price = estimate_price(mileage, theta0, theta1)
    print(f"Estimated price: {price:.2f}")


if __name__ == "__main__":
    main()
```

- Загружает \(\theta_0\), \(\theta_1\) из `theta.csv`.
- Спрашивает у пользователя пробег в километрах.
- Считает стоимость по формуле.
- Выводит результат с точностью до двух знаков после запятой.

1. **Обучаем модель:**

   ```bash
   python3 train.py
   ```

   На экране увидишь что-то вроде:

   ```text

   theta0 = 8500.1234
   theta1 = -0.02890000
   ```

   А в файле `theta.csv` появится строка с этими значениями.

2. **Делаем предсказание:**

   ```bash
   python3 predict.py
   ```

   Пример диалога:

   ```text
   Enter car mileage (km): 100000
   Estimated price: 5610.00
   ```

   Здесь 100000 — пробег, который ты ввёл, а 5610.00 — предсказанная цен
   а по обученной модели.
